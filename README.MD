# SkillBridge API: Educational Management & Monitoring System

SkillBridge is a vocational training management backend built with FastAPI. It features a sophisticated dual-layer security model and a relational database schema designed to track multi-institution educational progress.

## 1. Live API & Deployment Notes

- **Base URL:** `http://localhost:8000` (local development)
- **Interactive Docs:** `http://localhost:8000/docs` (Swagger UI)

**Deployment Note:**  
This API is configured for local production environments using Uvicorn. For cloud deployment (for example AWS or Heroku), ensure `SECRET_KEY` and `MONITORING_API_KEY` are set through environment variables rather than `.env` files.

## 2. Local Setup Instructions

Follow these steps to get the environment running from scratch:

1. Clone or extract the submission folder.
2. Navigate to the project root:

```bash
cd submission
```

3. Install dependencies:

```bash
pip install -r requirements.txt
```

4. Initialize the database:

```bash
cd src
python seed.py
```

5. Run the API:

```bash
uvicorn main:app --reload
```

## 4. Sample Commands

### A. Obtain Standard Access Token
Use this for Trainer, Student, and Institution roles:

```powershell
$login = Invoke-RestMethod -Method POST -Uri "http://127.0.0.1:8000/auth/login" `
    -Headers @{"Content-Type"="application/json"} `
    -Body '{"email":"trainer0@test.com","password":"password"}'
# Token is now in $($login.access_token)
```

### B. Obtain Scoped Monitoring Token
Use this for the Monitoring Officer only:

```powershell
# Step 1: Login
$mo = Invoke-RestMethod -Method POST -Uri "http://127.0.0.1:8000/auth/login" `
    -Body '{"email":"officer@gov.com","password":"password"}' `
    -Headers @{"Content-Type"="application/json"}

# Step 2: Exchange with System Key
$scoped = Invoke-RestMethod -Method POST -Uri "http://127.0.0.1:8000/auth/monitoring-token" `
    -Headers @{"Authorization"="Bearer $($mo.access_token)"; "Content-Type"="application/json"} `
    -Body '{"key":"SKILLBRIDGE_ADMIN_2024"}'
```

### C. Create Batch
Trainer example:

```powershell
Invoke-RestMethod -Method POST -Uri "http://127.0.0.1:8000/batches" `
    -Headers @{"Authorization"="Bearer $($login.access_token)"; "Content-Type"="application/json"} `
    -Body '{"name":"Advanced AI Class"}'
```

## 5. Schema & Design Decisions

### Many-to-Many (`batch_trainers`)
Vocational training often requires multiple subject matter experts for one batch. Using an association table allows for flexible trainer assignments without duplicating batch data.

### `batch_invites` (Invite Tokens)
To prevent unauthorized enrollment, students cannot join a batch by ID. They must possess a cryptographically secure token generated by a trainer, which is invalidated upon use.

### Dual-Token Approach
To satisfy Task 2’s high-security requirement, Monitoring Officer access is decoupled from standard login. Even if an officer’s password is compromised, the `/monitoring/attendance` endpoint remains locked unless the secondary system API key is provided to generate a short-lived scoped token.

## 6. Implementation Status

### Fully Working
- JWT authentication and bcrypt password hashing.
- RBAC for all 5 roles: Institution, Trainer, Student, Officer, Manager.
- Dual-token exchange flow for monitoring data.
- Automated database seeding with relational integrity.
- Batch creation and student-to-batch mapping.

### Partially Done
- Student enrollment flow: the backend logic for invite validation is present, but the automated join endpoint is simulated via the seed script.

### Skipped
- Frontend interface: as per requirements, the focus remained on the robust CLI-runnable API.
- File uploads: attendance is tracked via JSON data rather than image or biometric uploads.

## 7. Future Improvements

If given more time, I would implement token blacklisting. Currently, JWTs are stateless and valid until expiration. Implementing a Redis-backed blacklist would allow for immediate logout capabilities and enhanced security if a Monitoring Officer’s device is reported lost or compromised.
